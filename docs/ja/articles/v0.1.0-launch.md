# GeminiVRM v0.1.0 リリースガイド

## まず把握してほしい変更点

- Gemini Live の音声は逐次チャンクで再生されるため、応答の途中からアバターの反応が始まります。
- 同梱の `public/Kiyoka.vrm` を基点に、ライブモデル・音声・system prompt をその場で調整できます。
- `Settings` → `Streaming` → `YouTube relay` から、受信コメントを GeminiVRM の既存チャットへ取り込めるようになります。
- 初回リリースとして、英語/日本語の docs 体系、QA inventory、公開向けリリース資産（ヘッダ画像含む）が揃っています。

## 初回公開前の最短確認手順

1. アプリを起動します。

```bash
npm install
npm run dev -- --hostname 127.0.0.1 --port 3100
```

2. Gemini API key などの基本設定を完了します。
3. YouTube 連携が必要な場合のみ `Settings` → `Streaming` → `YouTube relay` を開きます。
4. Google サインイン、配信選択、relay / auto-reply の有効化を実施します。
5. 実配信と `npm run e2e:smoke` で安定性を確認し、公開前の検証を終えます。

## リリース運用ノート

- YouTube の受信コメントは GeminiVRM のチャットへ取り込まれます。
- auto-reply は GeminiVRM 内部の返信であり、YouTube チャットへの投稿はしません。
- 配信者自身のコメントは無限ループ防止のため無視されます。

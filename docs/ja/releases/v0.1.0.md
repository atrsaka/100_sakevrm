## ハイライト

- ブラウザ完結の VRM チャットが Gemini Live ネイティブ音声に対応し、部分 transcript と音声ストリーミングを同じターン内で返せるようになりました。
- 16-bit PCM chunk を先行スケジュールすることで、返答全体の完了前から再生と lip sync が始まります。
- `public/Kiyoka.vrm` を同梱し、ローカル VRM 差し替え、live model と voice の切り替え、system prompt 編集、ローカル設定保存までアプリ内で完結します。
- idle motion は `default`、`stand`、`sway`、`wait` の組み込みプリセットと、取り込み VRMA の smoothing まで含めて運用できる状態になっています。
- `Settings` -> `Streaming` -> `YouTube relay` から、配信の選択、受信コメントのプレビュー、Gemini への自動応答を使う任意のライブ連携を有効化できます。

## ツールと配布

- smoke E2E により、Gemini の実キーがなくても起動、送信フロー、既知の fallback 挙動を確認できます。
- GitHub Actions と Pages workflow で、アプリ本体と VitePress docs を `.next-pages` へ束ねて配布できるようにしています。
- `npm run dev:all` と `npm run build:pages` で、ローカル開発と Pages 向け静的出力の流れをそろえています。

## ドキュメントとアセット

- getting started、usage、YouTube relay、architecture、deployment、troubleshooting、QA inventory を英日両対応の VitePress docs として整備しています。
- OGP 画像、vector favicon、docs ホームのヒーロー画像まで含めて、公開向けの見た目を整えています。
- 今回のリリースノートも英語版と日本語版を docs 内にそろえ、初回リリースからバージョン別の記録を残せるようにしました。

## 既知の制約

- Gemini API key は引き続きブラウザ側で入力する local-first 運用で、サーバー中継型の構成ではありません。
- マイク入力は Web Speech API に依存し、現状は `ja-JP` 固定です。
- YouTube relay は relay 開始後の新着コメントをアプリ内 Gemini チャットへ取り込む機能で、Gemini の返答を YouTube コメントへ投稿したり、映像配信自体を送出したりはしません。
- この草案作成時点では `v0.1.0` タグと GitHub Release をまだ作成していないため、公開日と live URL はここに記載していません。

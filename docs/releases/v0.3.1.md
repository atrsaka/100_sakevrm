# v0.3.1 Release Notes

> Status: Release docs for `v0.3.1`
> Compare basis: `v0.3.0..v0.3.1`
> Scope: Document grounding mode, high-quality VRM assets, large VRM loading fix, and ElevenLabs ConvAI audio timing stabilization.

## Highlights

- **Document mode** lets users upload PDF, plain text, or Markdown files and ground both character chat and podcast conversations on the uploaded content. The system enforces strict grounding rules — the LLM must only reference information present in the document and must not fabricate or extrapolate.
- **ElevenLabs ConvAI timing fix** resolves the "agent returned no audio" errors that occurred when ElevenLabs took longer than 600ms to produce the first audio chunk. A dedicated first-audio timer (8 seconds) now waits for the initial chunk, and the inter-chunk idle timeout was doubled from 600ms to 1200ms.
- **High-quality VRM assets**: Kiyoka and Yukito models replaced with higher-fidelity versions, and the background image updated to a higher-resolution variant.
- **Large VRM loading**: `Model.loadVRM()` switched from `GLTFLoader.load()` to `fetch()` + `loader.parse()` to avoid XHR truncation on files over 30MB.

## Document Mode

- New module `src/features/document/documentParser.ts` handles PDF (via `pdfjs-dist` loaded from CDN), plain text, and Markdown parsing in the browser with a 30,000 character cap.
- New module `src/features/document/documentPromptBuilder.ts` constructs grounding-aware system prompts for both chat and podcast flows, including explicit rules against hallucination.
- Podcast persona generation (`podcastPersonaDesigner.ts`) now accepts a `documentSummary` parameter so the generated show concept and host personas align with the document content.
- Settings UI exposes a document upload section with file type validation and a clear button.

## ElevenLabs ConvAI Stabilization

- `TURN_AUDIO_IDLE_MS` increased from `600` to `1200` ms — the inter-chunk silence threshold that signals a turn is complete.
- New `FIRST_AUDIO_TIMEOUT_MS = 8000` ms — a separate, longer timeout that only applies before the first audio chunk arrives. Once audio begins, this timer is cleared and the standard idle timer takes over.
- All timer cleanup consolidated into a single `clearTimers()` helper to prevent timer leaks on session close or error.
- `sendTurn()` now starts the first-audio timer immediately upon dispatching `user_message`, and clears it as soon as the first PCM chunk is received.

## VRM Loading

- `Model.loadVRM()` now fetches the full `ArrayBuffer` via `fetch()` before calling `GLTFLoader.parse()`, avoiding silent truncation of large VRM files by the Three.js XHR loader.

## Other Changes

- `introduction.tsx` simplified and fully localized to Japanese.
- `meta.tsx` title updated to "おしゃべりVRM".
- `podcastStage.tsx` viewer creation accepts `motionOffsetSeconds` to stagger avatar motion start between the two podcast hosts.
- OGP images (`ogp.jpg`, `ogp.png`) updated.

## Known Constraints

- Document mode truncates content beyond 30,000 characters. Image-only PDFs are not supported and will show an error.
- The ElevenLabs first-audio timeout of 8 seconds is a fixed value. Extremely slow network conditions or overloaded agents may still time out.
- Podcast mode remains turn-based and does not overlap both hosts speaking at once.
